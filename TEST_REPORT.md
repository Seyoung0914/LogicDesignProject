# Quine-McCluskey 알고리즘 테스트 보고서

본 문서는 `mcCluskey` 패키지의 Quine-McCluskey 알고리즘 구현체에 대한 테스트 과정 및 결과를 정리한 보고서입니다.

## 1. 테스트 환경
- **언어:** Java 17 (Target)
- **컴파일러:** `javac`
- **테스트 도구:** 별도 제작된 `McCluskeyTest.java` (JUnit 스타일의 자동화 테스트 스크립트)
- **주요 구성 파일:**
  - `Main.java`: 프로그램 실행 진입점
  - `McCluskey.java`: 인터페이스 정의
  - `McCluskeyImpl.java`: 알고리즘 핵심 로직 구현체
  - `PI.java`: Prime Implicant 데이터 모델
  - `McCluskeyTest.java`: 자동화 테스트 스위트

## 2. 테스트 과정
테스트는 `ByteArrayInputStream`을 사용하여 사용자의 키보드 입력을 시뮬레이션하고, `ByteArrayOutputStream`을 통해 콘솔에 출력된 결과 문자열을 캡처하여 기대값과 비교하는 방식으로 진행되었습니다.

### 테스트 단계:
1. 소스 코드 패키지화 및 컴파일 (`package mcCluskey` 적용)
2. 각 테스트 케이스별 입력 시나리오 구성
3. 프로그램 실행 후 최종 출력 결과(Boolean Expression) 추출
4. 기대값(Expected)과 실제값(Actual) 비교 검증

## 3. 테스트 케이스 및 결과

## 4. 종합 평가
- **정확성:** 기본적인 Minterm 병합부터 Don't Care 처리, Essential PI 추출 및 차트 축소 로직이 의도한 대로 동작함을 확인하였습니다.
- **안정성:** Full 1(결과가 1) 또는 Full 0(결과가 0)과 같은 에지 케이스에서도 프로그램이 비정상 종료되지 않고 정확한 식을 산출합니다.
- **결론:** 현재 구현된 `McCluskeyImpl` 클래스는 논리 설계 프로젝트에서 요구하는 Quine-McCluskey 알고리즘의 표준 동작을 완벽히 수행합니다.

## 5. 향후 개선 제안 (요약)
